---
name: Drizzle lib rebuild after schema changes
description: After adding columns to the DB schema, the lib declarations must be rebuilt before leaf packages can see the new types.
---

## Rule
After any change to `lib/db/src/schema/*.ts` (adding columns, new tables), run:

```bash
pnpm run typecheck:libs
```

before running `pnpm --filter @workspace/api-server run typecheck`. Without this, the api-server still sees stale `.d.ts` declarations and throws "Property 'newColumn' does not exist on type …" errors.

**Why:** `lib/db` is a composite TypeScript project. Its `.d.ts` declarations are cached and only regenerated by `tsc --build`. The leaf artifact's typecheck does not trigger this rebuild automatically.

**How to apply:** Any time DB schema columns are added or removed, run typecheck:libs first as part of the workflow.
